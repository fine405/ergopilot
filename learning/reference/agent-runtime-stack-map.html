Agent Runtime 技术栈速查图

用于开发和面试前快速复习。打印版会自动隐藏交互元素。

四层职责

层核心问题主要技术输出
体验层用户看见并控制什么TanStack Start、shadcn、AI Elements意图、审批、进度、证据
认知层用户到底想做什么Mastra、LLM、Tools、MCP经 schema 校验的 TaskSpec
控制层任务如何可靠完成Hono、Workflows、Durable Objects持久 run 和有序 command
设备层物理动作是否真实完成Tauri、Rust、Tokio、SQLite经验证的设备状态和事件

一条命令的生命线

  1. 用户目标进入 UI。
  2. Agent 生成 TaskSpec。
  3. Schema validator 拒绝非法结构。
  4. Policy engine 返回 allow、approval 或 deny。
  5. Workflow 创建持久 run。
  6. Durable Object 把命令投递给 station。
  7. Rust runtime 校验 state version、grant 和 expiry。
  8. 命令先写 journal，再执行副作用。
  9. Simulator 发出 progress 和 terminal event。
  10. Runtime 读取状态并验证，Workflow 才能完成。

核心词汇

词一句话定义不要误解为
Agent用模型、上下文和工具迭代解决开放任务的系统聊天框
Agent loop观察 → 决策 → 工具 → 新观察的循环可靠工作流本身
Tool提供给模型的 schema 化操作任意可执行代码
Capability含风险、条件、超时、取消和验证的系统能力只带名字的函数
MCPAI 应用发现和调用工具、资源、提示的协议权限或恢复系统
Workflow可持久化的确定性步骤和状态迁移模型自由推理
State machine合法状态及合法迁移的显式模型一组布尔变量
Idempotency重复请求不会重复产生副作用请求只发送一次
Journal执行前后追加记录的持久事件日志普通 console log
Reconciliation重新观察事实并修正系统认知无条件 retry
Digital twin可查询、可执行、可注入故障的设备状态模型只有动画的 3D 模型
Eval对 Agent 行为的可重复质量测量只看 demo 感觉

超时后的决策

第一问：真实状态可读吗？

目标已达到 → 记录成功；完全未变化 → 用同一个 command identity 做有界重试；部分变化或不可读 → 停止、suspend 或人工介入。

五个危险信号

LLM 直接给电机参数

缺少语义能力和安全 envelope。

超时立即再执行

可能重复真实副作用。

聊天消息是任务真相

消息可以重生成，任务状态不可以。

收到 ACK 就算完成

应读取设备状态验证结果。

只用 LLM-as-a-Judge

安全性必须由确定性测试保证。
